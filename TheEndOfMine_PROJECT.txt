1. บทนำ
The End of Mine เป็นเกมเอาตัวรอดแนว Survival Odyssey แบบ text-based ที่พัฒนาด้วย .NET MAUI สำหรับ Android, iOS และ macOS ผู้เล่นรับบทเป็นผู้รอดชีวิตในโลกหลังหายนะ ต้องบริหารทรัพยากร รักษาค่าสถานะ และตัดสินใจจากเหตุการณ์ที่เกิดขึ้นระหว่างการออกสำรวจ
วัตถุประสงค์ของแอป
• ให้ผู้เล่นฝึกการวางแผนและตัดสินใจภายใต้ทรัพยากรจำกัด เช่น เลือกว่าจะออกสำรวจ พักผ่อน ใช้ไอเทม หรือประหยัดของไว้ใช้ในเหตุการณ์ถัดไป
• สร้างประสบการณ์เกมเนื้อเรื่องที่เล่นซ้ำได้ โดยรอบเกมใหม่สามารถสร้าง Chapter, เหตุการณ์, ตัวเลือก และไอเทมเริ่มต้นใหม่ด้วย LLM หรือระบบ fallback ภายในแอป
• แสดงตัวอย่างการสร้างแอปเกมมือถือด้วย .NET MAUI, C#, local save, JSON asset และระบบเชื่อมต่อ API ภายนอก
ภาพรวมการทำงานของแอป
• ผู้เล่นเปิดแอป เข้าหน้าเริ่มเกม เลือกตัวละครชาย/หญิง และกรอกชื่อผู้รอดชีวิต
• ระบบสร้างเนื้อเรื่องเริ่มต้นและไอเทมเริ่มต้น จากนั้นให้เลือกความยาก Easy, Normal หรือ Hard
• หน้าหลักแสดงค่าสถานะ ได้แก่ HP, อาหาร, น้ำ, ความเหนื่อย, เสียง และการติดเชื้อ พร้อมเวลาในเกม
• ผู้เล่นกด GO OUTSIDE เพื่อเจอเหตุการณ์ เลือกทางเลือก แล้วรับผลลัพธ์ต่อค่าสถานะ ไอเทม และเส้นเรื่อง
• ผู้เล่นเปิด INVENTORY เพื่อดูหรือใช้ไอเทม และใช้เมนูเพื่อหยุดเกม บันทึกเกม หรือเปิด/ปิดเสียง
• เกมจบเมื่อ HP เหลือ 0 หรือเล่นครบ Chapter สุดท้าย โดยแสดงสาเหตุการตายหรือฉากจบตามเส้นเรื่อง
ประโยชน์ของแอป
• ผู้เล่นได้ฝึกคิดเชิงกลยุทธ์จากการบริหารทรัพยากรและรับผลของทางเลือก
• ระบบสุ่มและ LLM ช่วยเพิ่มความหลากหลาย ทำให้การเล่นแต่ละรอบไม่เหมือนเดิม
• ระบบบันทึกเกมช่วยให้ผู้ใช้กลับมาเล่นต่อได้ และระดับความยากทำให้เหมาะกับผู้เล่นหลายรูปแบบ
กลุ่มผู้ใช้เป้าหมาย
• ผู้ใช้มือถือที่ชอบเกมเอาตัวรอด เกมเนื้อเรื่อง และเกมตัดสินใจ
• นักศึกษาและผู้พัฒนาแอปที่ต้องการศึกษาตัวอย่างเกมบน .NET MAUI
• ผู้เล่นที่ต้องการเกมภาษาไทยที่เล่นเป็นรอบสั้น ๆ และมีระบบ save/load ในเครื่อง

2. คู่มือการใช้งานหน้าจอ
2.1 หน้า Splash / Title

ภาพที่ 1 หน้า Splash และ Title ของแอป
เมื่อเปิดแอป ระบบจะแสดงหน้า Splash ประมาณ 3 วินาที จากนั้นนำผู้ใช้ไปยังหน้าเริ่มเกม ภายในหน้า Intro จะมีข้อความ TAP TO START ให้ผู้ใช้แตะเพื่อเข้าสู่หน้าสร้างตัวละคร
2.2 หน้าเลือกตัวละครและกรอกชื่อ

ภาพที่ 2 หน้าเลือกตัวละคร
• ผู้ใช้เลือกตัวละครชายหรือหญิง โดยการแตะตำแหน่งตัวละครบนหน้าจอ
• กรอกชื่อผู้รอดชีวิตในช่องชื่อ จำกัดความยาวสูงสุด 8 ตัวอักษร
• กดปุ่ม START เพื่อเริ่มสร้างเกมใหม่
• ถ้าไม่กรอกชื่อ แอปจะแสดงแจ้งเตือนให้กรอกชื่อก่อนเริ่มเกม
• ระบบไม่มีการ Login และไม่มี User/Password สำหรับผู้ใช้
2.3 หน้ากำลังสร้างโลก

ภาพที่ 3 หน้า Loading ระหว่างสร้าง Chapter แรก
หลังจากกด START แอปจะสร้างข้อมูลรอบเกมใหม่ ได้แก่ Chapter 1, เหตุการณ์เริ่มต้น, ตัวเลือกในเหตุการณ์ และไอเทมเริ่มต้น 3 ชิ้น หากกำหนด API key ไว้ ระบบจะเรียก LLM เพื่อสร้างเนื้อเรื่อง หากไม่มี key หรือเรียก API ไม่สำเร็จ แอปจะใช้ข้อมูล fallback จาก story_tree.json, events.json หรือระบบสุ่มภายในแอปแทน
2.4 หน้าเลือกระดับความยาก

ภาพที่ 4 หน้าเลือกระดับความยาก
ระดับ
ผลต่อการเล่น
Easy
ค่าสถานะลดช้าที่สุด ผลเสียจาก event ถูกลดลง และสามารถกลับไป daily checkpoint ได้
Normal
ค่าความยากสมดุล เหมาะกับการเล่นทั่วไป และยังมี daily checkpoint
Hard
ค่าสถานะลดเร็ว ผลเสียแรงขึ้น และไม่มี respawn แบบ daily checkpoint เมื่อเสียชีวิต

ผู้ใช้แตะเลือก Easy, Normal หรือ Hard จากนั้นกดปุ่ม Confirm เพื่อเข้าสู่หน้าหลักของเกม
2.5 หน้าหลักระหว่างเล่นเกม

ภาพที่ 5 ฉากพื้นหลังของหน้าหลัก โดย UI เกมจะซ้อนอยู่ด้านบน
• แถบ HP แสดงพลังชีวิต ถ้าเหลือ 0 เกมจะจบ
• แถบอาหารและน้ำลดลงตามเวลาหรือผลของเหตุการณ์ หากลดต่ำมากจะเริ่มกระทบ HP
• แถบความเหนื่อยเพิ่มขึ้นตามเวลาและการกระทำ ผู้เล่นสามารถลดได้ด้วย REST หรือ SLEEP
• แถบเสียงเพิ่มจากการกระทำที่ดัง เช่น ยิง ทุบ งัด หรือพังสิ่งของ เสียงสูงเพิ่มความเสี่ยงถูกโจมตี
• แถบการติดเชื้อเพิ่มจากการถูกกัด แผลสกปรก เลือด สนิม หรือสภาพแวดล้อมอันตราย และลดได้ด้วยยา/ผ้าพันแผล
• เวลาในเกมเดินแบบ 1 วินาทีจริง เท่ากับ 1 นาทีในเกม

2.6 หน้าเหตุการณ์

ภาพที่ 6 ตัวอย่างภาพเหตุการณ์ในเกม
• หน้าเหตุการณ์แสดงชื่อ Chapter, ชื่อเหตุการณ์, คำบรรยาย และภาพประกอบ
• แต่ละเหตุการณ์มีตัวเลือก 2 ทางเลือก ผู้เล่นเลือกได้เพียงครั้งเดียว
• บางตัวเลือกต้องมีไอเทม ใช้ไอเทม หรือทำให้ไอเทมเสีย durability
• หลังเลือก ระบบแสดงผลลัพธ์และไอเทมรางวัลถ้ามี จากนั้นกด ไปต่อ เพื่อกลับหน้าหลัก
• ผลของตัวเลือกสามารถเปลี่ยน HP, อาหาร, น้ำ, ความเหนื่อย, เสียง, การติดเชื้อ และ inventory
2.7 หน้า Inventory

ภาพที่ 7 ตัวอย่างไอเทมในระบบ Inventory
หน้า Inventory แสดงกระเป๋าสัมภาระเป็นตาราง 4 คอลัมน์ โดยค่าเริ่มต้นมี 16 ช่อง หากผู้เล่นได้รับไอเทมประเภทกระเป๋าหรือ container ที่เพิ่ม carry capacity ระบบจะขยายจำนวนช่องเก็บของได้
• ด้านบนแสดงจำนวนช่องที่ใช้และน้ำหนักรวมของไอเทม
• แตะช่องไอเทมเพื่อดูชื่อ ประเภท ความหายาก น้ำหนัก durability และผลของไอเทม
• ปุ่ม ใช้ไอเทม จะแสดงเฉพาะไอเทมที่ใช้ได้ เช่น อาหาร น้ำ ยา ผ้าพันแผล หรือชุดปฐมพยาบาล
• เมื่อใช้ไอเทม ค่าสถานะจะเปลี่ยนทันที เช่น เพิ่มอาหาร เพิ่มน้ำ เพิ่ม HP ลดความเหนื่อย หรือลดการติดเชื้อ
• ไอเทมแบบ one-time use จะหายจากกระเป๋าหลังใช้ ส่วนอุปกรณ์ที่มี durability จะลดจำนวนครั้งที่ใช้ได้
2.8 เมนูเกม Save / Pause / Sound
• STOP ใช้หยุดเวลาในเกมชั่วคราว และปุ่มจะเปลี่ยนเป็น RESUME เพื่อกลับมาเล่นต่อ
• SAVE GAME ใช้บันทึกเกมด้วยตนเอง โดยระบบมี Auto Save ประมาณทุก 15 วินาทีระหว่างเล่นอยู่แล้ว
• SOUND ON/OFF ใช้เปิดหรือปิดเสียงเกม
2.9 ระบบ Chapter และความต่อเนื่องของเรื่อง

ภาพที่ 8 ตัวอย่างภาพระหว่างเข้าสู่ Chapter
เกมแบ่งเป็น 4 Chapter โดยแต่ละ Chapter มีเป้าหมาย 8 เหตุการณ์ เมื่อผู้เล่นเล่นครบเหตุการณ์ของ Chapter ปัจจุบัน ระบบจะสร้าง Chapter ถัดไปโดยอิงค่าสถานะ ไอเทม และประวัติการเลือกก่อนหน้า หากจำนวนเหตุการณ์ใน Chapter ยังไม่ครบ ระบบจะสร้างเหตุการณ์ต่อใน Chapter เดิมก่อน
2.10 หน้า Game Over / Ending

ภาพที่ 9 ตัวอย่างภาพฉากจบ
• ถ้า HP เหลือ 0 เกมเข้าสู่หน้า Game Over และแสดงสาเหตุการเสียชีวิต
• ถ้าเล่นครบ Chapter สุดท้าย เกมแสดงบทสรุปของเส้นเรื่องและภาพฉากจบ
• ใน Easy และ Normal หากมี daily checkpoint ผู้เล่นสามารถกลับไปเริ่มต้นวันนั้นได้
• ใน Hard mode ไม่สามารถ respawn จาก daily checkpoint ได้
• ปุ่มกลับหน้าเริ่มเกมจะลบ save เดิมและให้เริ่มรอบใหม่
3. ข้อมูลบัญชีผู้ใช้
แอป The End of Mine ไม่มีระบบสมัครสมาชิก ไม่มีระบบ Login และไม่มี User/Password ผู้เล่นใช้งานผ่านข้อมูลที่บันทึกในเครื่องเท่านั้น ข้อมูลที่ผู้ใช้กรอกมีเพียงชื่อ Survivor ในหน้าเริ่มเกม
4. การบันทึกข้อมูลและไฟล์สำคัญ
• SaveService และ GameDatabase ใช้บันทึก Survivor, GameState และ Inventory เพื่อให้กลับมาเล่นต่อได้
• เกมบันทึก checkpoint เมื่อเริ่มเกม เมื่อเลือก event เมื่อพัก/นอน เมื่อกด save และเมื่อออกจากหน้าหลัก
• ไฟล์ story_tree.json, events.json และ items.json เป็นข้อมูล fallback และฐานข้อมูลไอเทมของเกม
• ไฟล์ story_assets.json ใช้ map alias ของ chapter, event และ item ไปยังรูปภาพจริงในโปรเจค

5. ปัญหา ข้อจำกัด และสาเหตุ
ฟีเจอร์หลักของเกมสามารถใช้งานได้ครบในระดับเล่นได้จริง ได้แก่ สร้างตัวละคร เลือกระดับความยาก ออกสำรวจ เลือกเหตุการณ์ ใช้ไอเทม บันทึกเกม และจบเกม อย่างไรก็ตามจากการตรวจโปรเจคพบข้อจำกัดที่ควรระบุในรายงานดังนี้
ข้อจำกัด/ปัญหา
สาเหตุและผลกระทบ
การสร้างเรื่องด้วย LLM ต้องใช้ API key
ถ้าไม่มี TYPHOON_API_KEY, OPENAI_API_KEY หรือ LLM_API_KEY ระบบจะ fallback ไปใช้ข้อมูลภายในแอป จึงยังเล่นได้ แต่เนื้อเรื่องอาจไม่หลากหลายเท่าการเรียก LLM จริง
การสร้างภาพด้วย Gemini ต้องใช้ GEMINI_API_KEY
ถ้าไม่ตั้งค่า key แอปจะใช้รูปจาก asset catalog ที่มีอยู่ หรือเก็บ image prompt ไว้ก่อน จึงอาจไม่มีภาพ generated ใหม่ในบางรอบ
หน้า Difficulty ใช้ภาพพื้นหลัง female_difficulty_select ใน XAML
แม้เลือกตัวละครชาย ระบบ gameplay ยังบันทึกเพศได้ถูกต้อง แต่ภาพพื้นหลังหน้าเลือกระดับความยากในเวอร์ชันนี้ยังไม่ได้สลับตามเพศ
ส่วน Skills บนหน้าหลักยังเป็น label แสดงผล
Model มี SkillSet แต่ GameEngine ยังไม่ได้เพิ่มระบบ EXP/level up ให้ครบ จึงควรนับเป็นส่วน UI เตรียมไว้สำหรับพัฒนาต่อ
ข้อความ Wioating EXP ในหน้าหลักสะกดผิด
เป็นข้อความ UI ใน MainPage.xaml ควรแก้เป็นค่าทักษะที่ต้องการ เช่น Water/Waiting/Survival ก่อนส่งเวอร์ชันสมบูรณ์
